# Security

If you discover a security issue, please report it responsibly.

**Do not open a public issue.** Send details to the maintainers (e.g. via GitHub Security Advisories or the repository owner’s contact method). Include steps to reproduce and impact where possible.

We will respond as quickly as we can and ask that you allow time for a fix before any public disclosure.

**Do not commit `.env` or any secrets.** This repository’s `.gitignore` excludes `.env`. Credentials belong only in environment variables or secure secret stores, never in the repo.
